[WMI] 1. Support WMI in UEFI BIOS

Posted on 2024-10-11 In WMI
15k 14 mins.

This document records how to support WMI in UEFI BIOS.

1. Introduce

2. Porting Guid

  • You can only choose one method bewteen Method 1 and 2, can not use both method at the same time

    Method 1: Transfer .mof to .dll and Sign up into register list

    • Write your .mof file, Below is used for reference

    • Open WmiStudy.sln by Visual Studio and compile .mof to .dll

    • Sign up .dll into register list

    • Declare declare “PNP0C14” into DSDT or SSDT then update the new UEFI BIOS Rom to your test mechine, Below is used for reference

    Method 2: Transfer .mof to .x and save the value of .x into ASL CODE

    • Write your .mof file, Below is used for reference

    • Compile .mof to .bmf in CMD

    • Compile .bmf to .x in CMD
    • Declare “PNP0C14”, Copy the value in the .x to Name(WQBA, Buffer()) in the asl code into DSDT or SSDT then update the new UEFI BIOS Rom to your test mechine, Below is used for reference

3. Debug

  • Reboot your test mechine if you finished part 2

  • Open WMICodeCreator.exe, you will see the class: A_SamTest_MULong under Namespace: root\WMI in the bar: Query for data from a WMI class, you can debug under the bar: Execute a method or use Visual studio to open projects in the Application\A_SamTest_MULong(Corresponds to the method 1 or 2 you are using)


  • If your WMI class does not appear in WMICodeCreator.exe, please follow the steps below in CMD, then reboot your test mechine
