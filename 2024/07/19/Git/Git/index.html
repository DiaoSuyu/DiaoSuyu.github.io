1. Merge local Git repository into a specific Gerrit repository

This topic is used to record how to merge a local Git repository(with multiple commit commit) into a specific Gerrit repository.

  • Jump into your Gerrit directory

  • Displays the commit history of the current branch, including author, date, and commit information

  • Create and switch to a new branch named temp-branch, pointing the branch to the specific commit ID(in your Gerrit directory)

  • Add the /d/Project/MCU_I2C_Driver/CRB directory as a remote repository of /d/Project/SAM_TEST called temp-repo

  • Gets the latest commit record from the temp-repo remote repository, but it doesn’t automatically merge into the current branch

  • Merge the contents of the temp-repo/master branch into the current branch (temp-branch)

  • Displays the status of the current working directory and staging area

  • Displays the commit history of the current branch, including author, date, and commit information

  • Push the temp-branch branch to the master branch of the remote repository origin

  • Delete the local temp-branch branch

3. Git stash

3.1 Temporarily save current changes

  • git stash push: temporarily store the changes on the stash stack

  • -m: add a description for this stash

  • changes file path: the path(s) of the files you want to stash

3.2 Check the stash list

  • git stash list: use this command to check all stash object

3.3 Recover changes from the stash

  • git stash pop: recover the stash and remove it from the stash list

  • git stash apply: recover the stash and keep it in the stash list

3.4 Delete a stash recovered by git stash apply

  • git stash drop: delete a stash (for example one you already recovered with git stash apply)
