[Windbg] Blue screen debug method

Posted on 2025-06-11 In Windbg
14k 12 mins.

This topic is used to record How to use windbg for KMDF Driver blue screen debug based on the Analysis of Dump File.

1. Grab the blue screen log

  • Find out the blue screen log under this location C:\WINDOWS\Minidump
    such as 061125-28265-01.dmp

2. Debug with Windbg

2.1 Load the log

  • Open Windbg, select File > Open Crash Dump
  • Wait for loading
  • Enter the analysis command !analyze -v
  • Specify the path to the .sys and .pdb (usually these two together)
  • Load your driver(.sys)

  • Enter the analysis command !analyze -v again

    • This time, you can see the specific code location of the error
